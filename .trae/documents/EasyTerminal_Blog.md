# 打造次世代终端：我是如何开发 EasyTerminal 的？

在 AI 和 Agent 工具爆发的今天，命令行不再只是简单的字符输入框，而是我们与 AI 助手沟通的桥梁。作为一个经常使用 Claude Code、Cursor CLI 等 Agent 工具的开发者，我发现传统的终端模拟器（如 iTerm2、Mac Terminal）在处理复杂的 TUI（Text User Interface）交互时显得力不从心。

最典型的痛点就是：**当 AI 在终端里抛出一个带有多个选项的选择题时，你必须用键盘的上、下方向键去小心翼翼地移动光标，然后再按下回车确认。** 这种体验在现代 GUI 时代显得非常割裂。

为了解决这个问题，并打造一个专为 AI Agent 时代设计的终端，我开发了 **EasyTerminal**。

## 核心功能与亮点

EasyTerminal 并不是在重复造轮子，而是在传统的 `xterm.js` 基础上，赋予了它“视觉感知”的能力。

### 1. “灵动岛”交互：让终端选择题变成原生 UI
这是我最喜欢的一个功能，也是开发中最具挑战性的部分。

当终端底层的命令行程序（例如 Inquirer.js、Claude CLI）输出需要用户选择的菜单时，EasyTerminal 会自动拦截这些信息，并在应用顶部弹出一个类似 macOS “灵动岛” 的原生悬浮窗口。

你可以直接用鼠标点击选项，或者用鼠标悬停查看多行长文本描述。点击后，系统会在后台自动向终端注入对应的键盘指令（例如按下三次下方向键，再按回车）。这彻底改变了在终端里做选择题的体验！

### 2. 现代化的工作流与文件管理
- **多会话管理**：左侧边栏支持创建多个独立的终端实例（Tab），支持重命名，方便你同时运行前端服务、后端服务和 AI Agent。
- **内联代码编辑**：内置了微软的 Monaco Editor。双击右侧资源管理器中的文件，它会直接在终端上方弹出一个玻璃态的编辑器面板，支持语法高亮和快速保存。
- **Markdown 实时预览**：对于 `.md` 文件，不仅支持纯文本编辑，还支持类似 GitHub 风格的 Markdown 实时渲染预览。
- **对话内容一键导出**：遇到有价值的 AI 对话或长段日志，只需点击终端顶部的 `[MD]` 或 `[PDF]` 按钮，就能将当前终端的全部内容干净地导出。

## 技术实现揭秘

作为一个基于 **Electron + React + TailwindCSS** 构建的桌面应用，背后其实有很多有趣的工程实现。

### 终端底层：node-pty + xterm.js
EasyTerminal 的核心是一个真实的终端环境。我在 Electron 的 Node.js 进程中使用了 `node-pty` 来衍生（spawn）真正的操作系统 Shell（如 `zsh` 或 `bash`），并通过 IPC 将数据流管道传输到渲染进程。

在渲染进程，使用 `xterm.js` 及其 `canvas` 插件将字符绘制到屏幕上。这保证了哪怕是高频的日志输出，依然能保持 60fps 的流畅度。

### 状态机与 TUI 拦截机制
灵动岛的魔法是如何实现的？其实这是一个基于 `xterm.js` Buffer 轮询和防抖（Debounce）的复杂状态机。

1. **Buffer 解析**：每隔 150 毫秒，渲染进程会读取终端当前屏幕的最后几行文本。
2. **正则匹配**：系统会扫描带有 `>`、`❯` 或 `●` 等指示符的行，以此作为“当前选中项”的锚点。
3. **选项提取**：向上回溯找到问题（Prompt），向后扫描缩进一致的文本块，合并成一个完整的选项列表。
4. **模拟按键注入**：当用户在灵动岛点击某个选项时，通过计算目标选项与当前高亮选项的索引差，向 `node-pty` 发送 `\x1b[A`（上）或 `\x1b[B`（下）指令，再发送 `\r`（回车），完美欺骗底层命令行程序。

### Electron 原生特性深度运用
- **透明窗口与无边框设计**：去掉了系统默认的标题栏，使用了 `backdrop-blur` 和自定义的拖拽区（`-webkit-app-region: drag`），实现了类似 macOS 原生应用的磨砂玻璃质感。
- **PDF 无头导出**：在导出 PDF 时，并没有使用前端容易排版错乱的 `jspdf`，而是在主进程中隐式创建了一个不可见的 `BrowserWindow`，注入格式化好的 HTML 代码，然后调用原生的 `webContents.printToPDF()` 方法，生成了完美的高清 PDF。

## 结语

EasyTerminal 是一次将“命令行的高效”与“现代 GUI 的优雅”结合的尝试。目前项目还在持续迭代中，如果你也是一个重度终端用户，或者正在开发自己的 AI CLI 工具，欢迎来体验一下这种全新的交互方式！
